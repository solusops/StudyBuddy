Architecture: The Aegis Augmented Teaching Loop

1. Core Concept: The "Translator" Model

Gemma 4 is not used as a source of truth. Instead, it is used strictly as a Cognitive Translator.
The source of truth is either the student's uploaded curriculum (PDFs, Word documents, Code, Images) OR dynamically fetched Open Educational Resources (OER) ingested into an open-source RAG microservice (e.g., Open Notebook, RAGFlow, or local ChromaDB). The AI fetches the dense facts and translates them into the student's selected familiarity level.

2. The Familiarity Profiles (System Modifiers)

Before the loop begins, the student selects a familiarity level. This dynamically alters the Tutor Agent's system prompt regarding vocabulary, analogy usage, and mathematical rigor, as well as dictating which OER databases the system queries.

ELI5 (Explain Like I'm 5):

Prompt Directive: Use purely sensory analogies (water, cars, building blocks). Zero mathematical formulas. Maximum 2-syllable vocabulary where possible.

Output Expectation: Focuses heavily on the concept and intuition.

OER Source: CK-12 (Primary Education) APIs or generalized encyclopedias.

High School / College:

Prompt Directive: Introduce standard terminology but immediately define it. Use relatable real-world applications. Include basic algebraic equations if necessary.

OER Source: OpenStax API (https://openstax.org) for peer-reviewed textbooks, Project Gutenberg API (https://gutenbergapi.com) for classic literature.

Graduate:

Prompt Directive: Assume baseline domain competence. Do not waste tokens defining standard terms. Focus on edge cases, methodology, and complex interactions.

OER Source: OpenAlex (https://openalex.org) and CORE API (https://core.ac.uk) for peer-reviewed academic papers.

Expert:

Prompt Directive: Pure data synthesis. Focus on literature gaps, highly technical specifications, and raw mathematical/algorithmic proofs. Zero analogies.

OER Source: arXiv API for cutting-edge preprints and OpenAlex.

3. The Curriculum Initialization Pipeline (Zero-Content Scenario)

If a student requests to learn a topic from scratch without providing any documents, the system must build a "Synthetic RAG" database on the fly before teaching begins.

Syllabus Blueprinting: The Brain Agent generates a JSON array of sub-topics that make up a standard curriculum for the requested subject (e.g., requesting "Quantum Mechanics" yields Wave-Particle Duality, Planck's Constant, etc.).

OER Targeted Fetching: * The Python backend queries the appropriate OER API based on the Familiarity Profile.

Instead of downloading a massive monolithic PDF, the backend fetches the specific HTML or JSON chapters matching the blueprint.

Background Vectorization:

The fetched text is processed through LangChain's TextSplitters (with appropriate overlap to preserve mathematical formulas).

It is vectorized using a fast local embedding model (e.g., nomic-embed-text or bge-small-en-v1.5) and inserted into the local in-memory ChromaDB.

UI Note: During this 5-15 second process, the React frontend displays an interactive "Constructing Curriculum..." tree animation.

Local Archiving: The downloaded OER chapters are saved locally as .md files in the student's Custom Workspace folder for offline reading.

4. The Grounded Teaching Loop (Step-by-Step)

When a student arrives at a node (e.g., "Neural Networks") and requests to learn it, the system executes the following strict pipeline:

Phase A: Knowledge Retrieval (The RAG Fetch)

The Brain Agent formulates a highly specific search query based on the active node.

The backend calls the API of the local RAG Microservice (Open Notebook/RAGFlow) or queries the local ChromaDB populated in the Initialization Pipeline.

The RAG service returns the raw, verbatim text chunks from the uploaded PDFs/documents or OER textbooks, complete with citations (e.g., "Source: OpenStax Biology 2e, Section 4.2").

Phase B: Translation & Generation (The Tutor Agent)

The Tutor Agent (Gemma 4) receives the raw chunks and the student's Familiarity Profile. It generates a three-part lesson:

The Anchor: A conceptual introduction tailored to the familiarity level.

The Grounded Truth: The actual facts, rewritten for readability, but strictly containing inline citations pointing back to the uploaded curriculum or fetched open books.

The Interactive Visual: (Optional) Generation of HTML5/CSS code to visualize the concept interactively.

Phase C: YouTube Curation (The Infinity Wiki Agent)

Simultaneously, the Infinity Wiki Agent executes a search to find supplementary human lectures.

It queries the YouTube Data API with the topic AND the familiarity modifier (e.g., "Neural Networks explanation for kids" vs "Neural Networks advanced backpropagation calculus").

It pulls the top 3 video transcripts, reads them rapidly, and selects the one that best matches the syllabus context.

The video URL is appended to the lesson payload.

Phase D: Evaluation (Active Recall)

The student cannot progress until they pass a dynamically generated test. The test questions are generated only from the retrieved curriculum/OER chunks to ensure fairness.

5. The Output: Markdown Summarization

To ensure the student has a persistent artifact of their studying (rather than losing knowledge in a scrolling chat window), the system automatically compiles the session into a single, cohesive Markdown file saved directly to their local folder.

Schema of [Topic]_Summary.md:

Title & Familiarity Level: (e.g., Action Potentials - High School Level)

Core Concepts: The translated explanations from the Tutor Agent.

Source Citations: A dedicated block linking the concepts back to the exact pages of their uploaded PDFs or the specific OER textbook API endpoints (e.g., See OpenStax Anatomy, Ch. 3).

Curated Media: Embedded links to the YouTube lectures curated by the Infinity Wiki Agent.

Active Recall Log: A record of the questions the student failed during the session and the correct answers, tagged for review in the next session.

By outputting a structured Markdown file, the student builds a personalized, searchable textbook over time that is perfectly aligned with their specific curriculum.
